## 环境准备
- Node.js ≥ 18（Vite 6 需要），已安装 `npm`
- 操作系统：Windows（当前目录 `d:\积分制`）
- 项目脚本：见 `package.json:6-12`，包含 `dev/build/preview`

## 安装依赖
- 在项目根目录执行：`npm install`
- 使用 `npm`（仓库存在 `package-lock.json`）

## 配置环境变量
- 使用项目根目录的 `.env`（已检测到存在）
- 必填变量（缺失将报错，见 `src/lib/supabase.ts:3-8`）：
  - `VITE_SUPABASE_URL=<你的 Supabase 项目 URL>`
  - `VITE_SUPABASE_ANON_KEY=<你的 Supabase 公钥>`
- 注意：不在输出中展示具体值；确保为有效的 Supabase 项目配置

## 启动开发服务器
- 运行：`npm run dev`
- 默认地址：`http://localhost:5173/`（`vite.config.ts`未自定义端口）
- 我将在成功启动后提供可点击预览链接

## 验证与常用命令
- 构建产物：`npm run build` 后可用 `npm run preview` 本地预览
- 类型检查与代码质量：`npm run check`、`npm run lint`

## 可选：数据库迁移说明
- 若使用自建 Supabase 实例，参考 `supabase/migrations/` 下 SQL 脚本按顺序迁移
- 前端运行不要求本地数据库，只需可用的 Supabase 云项目与上述环境变量
